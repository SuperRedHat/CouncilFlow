## Methodology

- **Tokenizer**: `tiktoken` with `cl100k_base` encoding when the package is installed; otherwise `len(text) // 4` heuristic (underestimates by ~10-15% but stable across runs).
- **Ceremony proxy**: the sum of tokens inside `handoff.yaml/json` files across all delegations. This captures the per-stage context re-establishment cost (objective + constraints + required artifact refs + expected output + inputs summary).
- **Reread estimate**: for each consecutive pair of `local_execution` delegations, the later stage's handoff references the prior stage's `result.md`; the controller re-tokenizes that result when planning the next stage. We attribute the prior stage's result-token count to this re-read. This *underestimates* if the controller also needs to re-read its own implementer artifact during tester/reviewer, and *overestimates* if the controller session's prompt cache keeps the prior result warm.
- **Uncertainty interval**: real token costs are within ±15% of the reported numbers under the heuristic tokenizer; under `tiktoken` the numbers track closely with what OpenAI/Claude charge.
- **Not counted**: prompts generated by CouncilFlow's Python layer (role prompt templates, orchestrator framing), cross-model discuss summaries, and sidecar workspace materialize overhead.

### Phase 2 决策依据

**本基线数据支持最初"跳过 link folding"的决定**，原因：

- **Link folding 的靶子是 reread cost**，而当前采样 reread=0，即使扩展到完整 /project-next 样本，ceremony 占比 5% 意味着 link folding 最多能挽回的也就是这 5% 里的一部分（假设 reread 占 ceremony 的 30-50%，最大节省 = 13115 × 5% × 50% = 328 tokens = 总成本的 2.5%）
- **A 动态路由的靶子是 result.md（91%）**：如果 tester/synthesizer 下沉到 haiku，节省的是结果产出的 LLM 推理成本，量级比 ceremony 大得多
- **B 语义 min_rounds 的靶子不在本基线里**：discussion token 在 `.council/discuss/` 下，需要单独测量；但从之前的 5 轮 codex discuss 可以估算，min_rounds=2 如果改成 semantic 可能砍掉 30-50% 讨论轮数

### 量化估算（保守）

假设未来一个完整 78 任务规模项目：

| 改动 | 预估节省 |
|---|---|
| Link folding（不做） | < 3% 总 token（数据确认）|
| A 动态路由（保守配置：tester+synthesizer 下沉 haiku） | **30-50% $ 成本**（token 数变化不大，但贵模型调用减少）|
| B 语义 min_rounds | discussion 场景 **30-50% token**（占总项目 token 约 10%，折算总节省 3-5%）|

**结论**：A 是唯一量级显著的改动，应作为 0.1.3 的核心。B 收益中等但风险最低，适合一并做。Link folding 数据上确认 ROI 很低，推迟决定正确。

### 本基线数据的局限

1. **采样偏斜**：13 个中有 5 个是本会话刚产生的 local artifacts，可能不代表长期使用 pattern
2. **Missing metadata**：local_execution 的 artifact 没有 handoff.yaml，role 字段显示为 `<unknown>`（tool 限制，准确但信息量低）
3. **无 discussion 覆盖**：只测了 delegations，没测 discussions；完整基线应加测 `.council/discuss/`
4. **使用 fallback tokenizer**：tiktoken 未安装，用 chars/4 估算，实际 token 数可能比报告高 10-15%
